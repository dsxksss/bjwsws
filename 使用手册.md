# 抗体设计批量打分与序列优化 —— 使用手册

本手册对应抗体亲和力优化的完整流程：**初始突变位点选择 → 单点突变打分（五工具）→ 序列优化设计**。
除「一、初始突变位点选择」为人工步骤外，其余均由本目录的批量脚本驱动，基于 **wemol-python-sdk** 提交/回收 WeMol 作业。

---

## 0. 脚本清单与环境准备

### 0.1 脚本目录

所有脚本位于部署机的本目录（示例：`/data/.../beijing_weishengwusuo/`）。清单：

| 脚本 | 作用 | 所属步骤 |
|---|---|---|
| `prepare_inputs.py` | 由 PDB + 位点生成饱和突变列表 + 各模块作业清单 `tasks.csv` | 二.1 |
| `submit_batch.py` | 批量提交 `tasks.csv` 里的作业（断点续传、自动处理并发上限） | 二.2/3/4/5 |
| `fetch_batch.py` | 回收已完成作业结果，汇总成 `scores.csv` | 二.2/3/4/5 |
| `sfe_md.py` | SFE 的 MD 编排：PDB Mutation → MD → 抽帧 →（自动）rechain，断点续跑 | 二.SFE (1)(2)(3) |
| `sfe_prepare.py` | 由 WT/突变体构象帧生成 forward/reverse 的 Flex DDG 作业清单 | 二.SFE (4) |
| `sfe_fetch.py` | 回收 SFE 的 Flex DDG 结果 → 42 列宽表 →（调 step02c）出 `sfe_scores.csv` | 二.SFE (4)(5) |
| `sfe_split_frames.py` | （被 sfe_md 调用）多帧 PDB 拆单帧 | 内部 |
| `sfe_rechain.py` | （被 sfe_md 调用）GROMACS 帧 → Rosetta 兼容（链/编号/去氢/命名归一） | 内部 |
| `build_pipeline_input.py` | 把 `scores.csv` 转成 `run_pipeline` 就绪的 `single_point_scores.csv` | 二.5 |
| `_common.py` | 共享工具（登录、任务加载、结果解析），不单独执行 | — |
| `_crpca_download/CRPCA/local_pipeline/step02c_compute_sfe_from_flex_ddg.py` | 42 个 ddG → SFE 分数 | 二.SFE (5) |
| `_crpca_download/CRPCA/local_pipeline/run_pipeline.py` | 序列优化（突变枚举 + GP/MEI + Pareto） | 三 |

### 0.2 运行环境（两套 Python）

脚本分属两类依赖，建议用两个解释器（或一个装齐全部依赖的环境）：

| 用途 | 依赖 | 脚本 |
|---|---|---|
| 提交/回收 WeMol 作业 | `wemol-python-sdk` | prepare_inputs / submit_batch / fetch_batch / sfe_md / sfe_prepare / sfe_fetch |

> 提示：`sfe_fetch.py` 调 step02c 时可用 `--step02c-python` 指向装了 numpy/pandas 的解释器。

## 一、初始突变位点选择（人工）

> 本部分需人工完成，脚本不介入。

1. **抗原-抗体复合物结构**：先在 PDB 数据库查是否已有该抗原-抗体复合物（人工确认）；有则下载，没有则结构预测（Boltz2 / Protenix / AF3 web server）。
2. **单点突变位点确认**：借助 WeMol 模块辅助人工分析，得到待突变残基位点列表，例如：
   - `Extract Fv and Analyze Contacts`（cutoff 7.0 Å）找抗体链 contact 残基；
   - `Protein Evolution → Single Point Mutation` 分析重/轻链优势突变。
   - 结论示例：重链 31,53,55-56,104-113,116 号 + 轻链 31-36,38,55-56,59 号残基。

## 二、单点突变位点打分

> 以下均在部署机执行。约定复合物结构文件名 `ab-ag-complex.pdb`，重链 `M`、轻链 `N`、抗原 `S`。

### 1. 生成单点饱和突变列表 —— `prepare_inputs.py`

```bash
python prepare_inputs.py \
    --pdb ab-ag-complex.pdb \
    --muts M31,M53,M55-56,M104-113,M116,N31-36,N38,N55-56,N59 \
    --omit-aas CP \
    --out-dir prod \
    --modules flexddg,foldx,fep,abnativ
```

| 参数 | 说明 |
|---|---|
| `--pdb` | 复合物结构文件 |
| `--muts` | 待突变位点，格式 `链名+残基位置`，逗号分隔，区间用 `-`（位置是每条链从 1 开始的顺序编号） |
| `--omit-aas` | 排除的突变后残基类型，如 `CP` 排除突变为 C/P |
| `--modules` | 要打分的模块：`flexddg` / `foldx` / `fep` / `abnativ`（逗号分隔，可任意组合） |
| `--out-dir` | 输出目录 |
| `--limit` | （可选）只取前 N 个突变，测试用 |

**输出**（写入 `--out-dir`）：
- `single_mut_list.txt`：饱和突变列表（每行一个，如 `DM31L`，= 原残基+链+位置+突变残基）
- `tasks.csv`：各模块作业清单（submit_batch 直接吃）
- `Mut_Hchains.fasta` / `Mut_Lchains.fasta`：突变后整链序列（AbNatiV 用，自动判定重/轻链）

`single_mut_list.txt` 示例：
```
DM31L
DM31A
DM31G
...
TM53A
TM53D
...
```

### 2. Flex DDG / FoldX / FEP / AbNatiV —— 统一批量提交与回收

四个模块（Flex DDG、FoldX=Mutation Energy of Binding、Protein FEP、AbnatiV）共用同一套批量脚本，无需逐个手动操作。

**提交**（一次性提交 tasks.csv 全部作业）：
```bash
python submit_batch.py --config config.json --tasks prod/tasks.csv -o prod/submitted.jsonl
```
- 每个作业返回作业号，可在 WeMol 网页作业列表查看状态；
- 自动处理平台并发上限（达到上限自动等待重试）；
- 中途中断可重跑同一命令，**断点续传**（已提交的自动跳过）。

**回收 + 汇总**（等作业跑完后）：
```bash
python fetch_batch.py --config config.json --input prod/submitted.jsonl --output prod/scores.csv
```
- 下载已完成作业结果，汇总成总表 `scores.csv`；
- **可重复运行**——还有作业没跑完时本次只汇总已完成部分，剩余等下次；
- `--foldx-interface` 默认 `M_S+N_S`（抗体两条链各自与抗原界面之和 = 抗体-抗原结合能变化；排除内部 M_N 重轻链界面）。单界面用 `--foldx-interface M_S`；多界面求和用 `+` 连接。

`scores.csv` 默认即对齐序列优化所需格式（见二.5）。如只想单跑某模块，在二.1 的 `--modules` 里指定即可。

### 3. 结构波动估计（SFE）

SFE 需要每个突变 **21 forward + 21 reverse = 42 个 Rosetta Flex ddG**，再聚合成 1 个分数。
完整链路已脚本化（GROMACS MD → 构象 → FlexDDG → 聚合）：

**(1)(2)(3) MD 编排 + 抽帧 —— `sfe_md.py`**

对 WT 与每个突变体各跑一次 MD，从轨迹抽 21 个构象（1 初始 + 20 个 MD 帧，Skip 500 ps），并自动做 Rosetta 兼容化处理（还原链/编号、去氢、AMBER 残基/原子名归一）。断点续跑——MD 慢，反复运行直到全部 `extracted`。

```bash
python sfe_md.py --config config.json --pdb ab-ag-complex.pdb \
    --mut-list prod/single_mut_list.txt \
    --receptor-chain S --ligand-chain M,N \
    --sim-ns 10 --skip-ps 500 --n-frames 21 \
    --wt-frames-dir prod/frames_WT --frames-root prod/frames_mut --out-dir prod
# 反复运行同一命令，直到输出「全部目标已 extracted」
```

| 参数 | 说明 |
|---|---|
| `--receptor-chain` / `--ligand-chain` | MD PDB Prepare 的受体/配体链（如抗原 S / 抗体 M,N） |
| `--sim-ns` | MD 模拟时长（文档要求 10 ns） |
| `--skip-ps` | 抽帧间隔（500 ps） |
| `--n-frames` | 期望帧数（21） |

**(4) 生成 forward/reverse 的 Flex DDG 作业 —— `sfe_prepare.py` + `submit_batch.py`**

```bash
python sfe_prepare.py --mut-list prod/single_mut_list.txt \
    --wt-frames-dir prod/frames_WT --wt-tag WT \
    --mut-frames-root prod/frames_mut --n-frames 21 --out-dir prod
python submit_batch.py --config config.json --tasks prod/tasks_sfe.csv -o prod/sfe_jobs.jsonl
```
- Forward：WT 的 21 帧 + 正向突变（如 `DM31L`）
- Reverse：突变体的 21 帧 + 反向突变（如 `LM31D`，虚拟恢复 WT）
- 每突变 42 个 Flex DDG 作业。

**(5) 回收 + 聚合出 SFE 分数 —— `sfe_fetch.py`（内部调 step02c）**

```bash
python sfe_fetch.py --config config.json --input prod/sfe_jobs.jsonl --n-frames 21 \
    --wide-output prod/sfe_flex_ddg_wide.csv \
    --run-step02c --sfe-output prod/sfe_scores.csv \
    --step02c-python <装了numpy/pandas的python>
```
- 把每个作业的 ddG 归位到 42 列宽表（`mutation,chain,forward_01..21,reverse_01..21`）；
- 调 `step02c_compute_sfe_from_flex_ddg.py` 去异常值取平均：`SFE = (IQR均值(forward) − IQR均值(reverse)) / 2`，输出 `sfe_scores.csv`。

> 也可手动单跑 step02c：
> ```bash
> PYTHONPATH=_crpca_download/CRPCA python _crpca_download/CRPCA/local_pipeline/step02c_compute_sfe_from_flex_ddg.py \
>     --input prod/sfe_flex_ddg_wide.csv --output prod/sfe_scores.csv
> ```

### 4. 人源性打分（AbNatiV）

已并入二.1/二.2 的统一批量流程：`--modules abnativ` 会用突变后整链 FASTA（`Mut_Hchains.fasta`/`Mut_Lchains.fasta`）批量打分，`fetch_batch` 取 `AbNatiV V* Score` 总分（4 位小数）作为 `abnativ`（Humanness）列。无需单独手动跑 abnativ 脚本。

### 5. 汇总得分 → 序列优化就绪输入 —— `build_pipeline_input.py`

`fetch_batch` 的 `scores.csv` 已是序列优化所需的列格式：

```csv
mutation,chain,location,original_aa,mutant_aa,sfe,fep,rosetta_flex,foldx,abnativ
DM31L,M,31,D,L,,-8.123,-11.038,-1.07,0.9000
```

但 `scores.csv` 的 `location` 是 **PDB resSeq**，而 `run_pipeline` 需要 **master 序列线性位置**（重链+轻链拼接后的 1-based）。进入第三步前需经 `build_pipeline_input.py` 修正、并合并 SFE：

```bash
python build_pipeline_input.py \
    --scores prod/scores.csv \
    --pdb ab-ag-complex.pdb \
    --antibody-chains M,N \
    --sfe prod/sfe_scores.csv \
    --output prod/single_point_scores.csv
```

| 列 | 说明 |
|---|---|
| `mutation` | 突变名称，如 `DM31L` |
| `chain` | 突变所在链名（属于 `--antibody-chains`，如 M、N） |
| `location` | master 序列线性位置（从 1 开始） |
| `original_aa` / `mutant_aa` | WT 残基 / 突变残基 |
| `sfe, fep, rosetta_flex, foldx, abnativ` | 五工具原始分数 |

> `--sfe` 可省略（没跑 SFE 时该列留空，`run_pipeline` 会自动跳过该工具）。

## 三、序列优化设计 —— `run_pipeline.py`

基于五工具单点评分，做组合突变枚举与筛选，得到优势突变组合。

```bash
PYTHONPATH=_crpca_download/CRPCA python _crpca_download/CRPCA/local_pipeline/run_pipeline.py \
  --single-point-scores prod/single_point_scores.csv \
  --pdb ab-ag-complex.pdb \
  --antibody-chains M,N \
  --antigen-chains S \
  --output-dir prod/optimization_run \
  --bo-rounds 3 \
  --num-sequences-per-round 2000 \
  --mei-batch-size 50 \
  --min-mutations 1 \
  --max-mutations 8 \
  --final-quota 20 \
  --seed 0
```

| 参数 | 说明 |
|---|---|
| `--single-point-scores` | 二.5 的单点打分 CSV |
| `--pdb` | 初始复合物结构 |
| `--antibody-chains` | 重链,轻链名（逗号分隔，如 M,N） |
| `--antigen-chains` | 抗原链名 |

带默认值参数：

| 参数 | 默认 | 说明 |
|---|---|---|
| `--bo-rounds` | 3 | 循环枚举轮数 |
| `--num-sequences-per-round` | 2000 | 每轮候选突变数 |
| `--mei-batch-size` | 50 | 每轮 MEI 保留最佳突变数 |
| `--min-mutations`/`--max-mutations` | 1 / 8 | 组合突变点范围 |
| `--final-quota` | 20 | 最终推荐组合数 |
| `--gp-num-iters` | 500 | GP 训练迭代 |

**最终输出**（`--output-dir`）：

| 文件 | 内容 |
|---|---|
| `final_recommendations.csv` | 推荐突变组合、mutationHumanReadable、全长 sequence、GP 预测、五工具聚合分 |
| `final_recommendations.fasta` | 同上，FASTA 格式 |
| `pipeline_summary.json` | 运行摘要 |
| `mei_pool_all_rounds.csv` | 所有 BO 轮 MEI 候选合并 |
| `gp_model/` | 训练好的 GP 模型 |

## 附录 A：各模块对应的 WeMol 模块

| 打分列 | WeMol 模块 | 类型 |
|---|---|---|
| `rosetta_flex` | Flex DDG | 单元型（1 突变 1 作业） |
| `foldx` | Mutation Energy of Binding | 批量型（1 作业管全部） |
| `fep` | Protein FEP | 单元型 |
| `abnativ` | AbnatiV | 批量型（重/轻链各 1 作业） |
| `sfe` | PDB Mutation + Protein MD Simulation v2.1 + MD Trajectory v2 + Flex DDG | 见二.3 |

## 附录 B：已知约束与建议

- **GPU 瓶颈**：FEP、SFE（含 MD + Flex DDG）均吃 GPU。若计算节点 GPU 数量有限，这两项做全量饱和扫描非常耗时（FlexDDG 单作业按默认 nstruct 约半天）。建议：`foldx`/`abnativ` 全量跑；`flexddg`/`fep`/`sfe` 视算力对关键突变跑。
- **断点续传**：submit/fetch/sfe_md 均可重复运行，中断不丢进度。
- **多点突变**：`prepare_inputs` 对 FlexDDG/FEP 仅处理单点；FoldX 可含多点。
