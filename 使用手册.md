# 抗体设计过程

本流程对抗体进行亲和力优化：**初始突变位点选择 → 单点突变位点打分（五工具）→ 序列优化设计**。
除「一、初始突变位点选择」为人工步骤外，其余均由本目录的批量脚本驱动（基于 wemol-python-sdk 提交/回收 WeMol 作业）。

---

## 一、初始突变位点选择

### 1、抗原-抗体复合物结构

这部分需要人工手动进行。对需要优化亲和力的抗体，先获取抗原-抗体复合物结构：先查看 PDB 数据库中是否已有抗原-抗体复合物结构（人工确认），如有则直接下载使用；如没有，则通过结构预测的方式获得（Boltz2、Protenix，或 AF3 的 web server）。

### 2、单点突变位点确认

这部分需要人工进行，借助 WeMol 模块辅助，有多种方式可参考：

- 基于 1 中的抗原-抗体复合物结构，使用 WeMol 模块 **Extract Fv and Analyze Contacts** 分析抗体链中的 contact 残基（距离 cutoff 可设为 7.0 埃），作为需要进行单点突变的残基。
- 基于 1 中的抗原-抗体复合物结构，使用 WeMol 模块 **Protein Evolution** 中的 **Single Point Mutation** 功能，分析重、轻链的单点优势突变。

通过人工分析，得到需要进行突变尝试的残基位置列表。例如：经分析后，需要对重链的 31、53、55-56、104-113、116 号残基，及轻链的 31-36、38、55-56、59 号残基进行突变分析。

## 二、单点突变位点打分

后续所有步骤在部署了批量脚本的机器上执行，脚本位于部署目录 **`/data/<部署路径>/beijing_weishengwusuo/`**（请按实际部署目录填写）。

首次使用前，配置一次登录信息：复制 `config_example.json` 为 `config.json`，填入 WeMol 平台地址与账号：

```json
{
  "base_url": "对应的私有化部署好的 WeMol 平台地址",
  "user": { "Name": "你的WeMol账号", "Passwd": "你的密码" },
  "result_dir": "wemol_results"
}
```

> 约定复合物结构文件名 `ab-ag-complex.pdb`，重链 `M`、轻链 `N`、抗原 `S`。

### 1、生成单点饱和突变列表

使用脚本 **`prepare_inputs.py`** 生成饱和突变列表。如重链名称为 M、轻链名称为 N，执行：

```bash
python prepare_inputs.py \
    --pdb ab-ag-complex.pdb \
    --muts M31,M53,M55-56,M104-113,M116,N31-36,N38,N55-56,N59 \
    --omit-aas CP \
    --out-dir prod \
    --modules flexddg,foldx,fep,abnativ
```

执行参数说明：

| 参数 | 说明 |
|---|---|
| `--pdb` | 复合物结构文件 |
| `--muts` | 需要进行突变的残基位点，格式为：逗号分隔的「链名称+残基位置编号」（位置编号是每条链中从 1 开始的顺序编号），区间用 `-` |
| `--omit-aas` | 单点突变中希望排除的突变后残基类型，如 `CP` 表示排除突变为 C 和 P |
| `--modules` | 需要打分的模块，可选 `flexddg` / `foldx` / `fep` / `abnativ`（逗号分隔，可任意组合） |
| `--out-dir` | 输出目录 |
| `--limit` | （可选）只取前 N 个突变，测试用 |

执行后，会在输出目录下得到单点突变列表文件 `single_mut_list.txt`，并同时生成各模块的作业清单 `tasks.csv`，以及 AbNatiV 用的突变后整链序列 `Mut_Hchains.fasta` / `Mut_Lchains.fasta`（自动判定重/轻链）。`single_mut_list.txt` 内容示例（部分）：

```
DM31L
DM31A
DM31G
DM31V
DM31S
DM31E
...
TM53A
TM53D
TM53G
...
```

### 2、Rosetta Flex 计算突变 ddG ／ 3、FoldX 计算突变 ddG ／ 4、人源性打分 ／ 5、自由能微扰评估热稳定性（FEP）

Flex DDG、FoldX（Mutation Energy of Binding）、AbNatiV、Protein FEP 四个模块通过同一套批量脚本提交与回收，无需逐个手动操作。

**提交作业**（一次性提交 `tasks.csv` 里的全部作业）：

```bash
python submit_batch.py --config config.json --tasks prod/tasks.csv -o prod/submitted.jsonl
```

提交后每个作业会返回一个作业号，可在 WeMol 网页作业列表查看运行状态。脚本会自动处理平台并发上限（达到上限自动等待重试），中途中断可重跑同一命令断点续传。

**回收并汇总**（等作业全部跑完后）：

```bash
python fetch_batch.py --config config.json --input prod/submitted.jsonl --output prod/scores.csv
```

该命令会下载已完成作业的结果，并汇总成一张总表 `scores.csv`：第一列为突变名，其余列为各模块打分。该命令可重复运行——还有作业未完成时本次只汇总已完成部分，剩余等下次。

- AbNatiV 取 `AbNatiV V* Score` 总分（4 位小数）作为 `abnativ`（Humanness）列，无需单独手动跑 AbNatiV 脚本。
- FoldX 的 `--foldx-interface` 默认 `M_S+N_S`（抗体两条链各自与抗原界面之和 = 抗体-抗原结合能变化；排除内部 M_N 重轻链界面）。单界面用 `--foldx-interface M_S`；多界面用 `+` 连接求和。

`scores.csv` 内容示例：

```
mutation,chain,location,original_aa,mutant_aa,sfe,fep,rosetta_flex,foldx,abnativ
DM31L,M,31,D,L,,-8.123,-11.038,-0.04,0.9000
DM31V,M,31,D,V,,-7.456,-10.221,0.00,0.9000
```

> 说明：`scores.csv` 是**固定列结构**（五工具列始终都在）。本步只跑了 FoldX/AbNatiV/FlexDDG/FEP，
> 所以 **`sfe` 列此时为空**——SFE 在下面第 6 步单独计算，最终在第 7 步汇总（`build_pipeline_input.py`）时合并补齐。
> 同理，某列只有在第 1 步 `--modules` 里跑了对应模块才会有值（没跑的列留空，`run_pipeline` 会自动跳过该工具）。

如只想单独运行某个模块，在第 1 步的 `--modules` 参数里指定即可，如 `--modules flexddg`。

### 6、结构波动估计（SFE）计算

SFE 需要每个突变 21 forward + 21 reverse 共 42 个 Rosetta Flex ddG，再聚合成 1 个分数。完整链路已脚本化：

**(1) 获取单点突变对应的结构 + (2) 基于 MD 进行构象采样 + (3) 获取多构象**

使用脚本 **`sfe_md.py`** 一步完成：自动调用 PDB Mutation 模块生成突变体结构，对 WT 及每个突变体各跑一次 MD（默认 10 ns），再用 MD Trajectory (v2) 模块抽取 21 个构象（1 个初始 + 20 个 MD 帧，Skip Time = 500 ps），拆分成单帧 PDB（命名 `突变名_n.pdb`），并自动做 Rosetta 兼容化处理（还原链/编号、去氢、残基/原子名归一）。断点续跑——MD 较慢，反复运行同一命令直到全部 `extracted`：

```bash
python sfe_md.py --config config.json --pdb ab-ag-complex.pdb \
    --mut-list prod/single_mut_list.txt \
    --receptor-chain S --ligand-chain M,N \
    --sim-ns 10 --skip-ps 500 --n-frames 21 \
    --wt-frames-dir prod/frames_WT --frames-root prod/frames_mut --out-dir prod
```

**(4) Rosetta Flex 计算突变 ddG**

对于每个单点突变：
- 正向计算（Forward）：取 WT 复合物的 21 个构象，根据单点突变计算 ddG；
- 反向计算（Reverse）：取突变体复合物的 21 个构象，虚拟恢复为 WT 计算 ddG。

用 **`sfe_prepare.py`** 生成 forward/reverse 的 Flex DDG 作业清单，再用 `submit_batch.py` 提交：

```bash
python sfe_prepare.py --mut-list prod/single_mut_list.txt \
    --wt-frames-dir prod/frames_WT --wt-tag WT \
    --mut-frames-root prod/frames_mut --n-frames 21 --out-dir prod
python submit_batch.py --config config.json --tasks prod/tasks_sfe.csv -o prod/sfe_jobs.jsonl
```

每个突变得到 42 个 ddG 数值，汇总为宽表 CSV（`mutation,chain,forward_01..21,reverse_01..21`），示例：

```
mutation,chain,forward_01,...,forward_21,reverse_01,...,reverse_21
G112E,M,-1.6781,...,-1.8739,-0.9405,...,-0.3284
S32A,N,-1.3329,...,-1.043,-0.0655,...,-0.4527
```

**(5) 聚合出 SFE 分数**

用 **`sfe_fetch.py`** 回收 Flex DDG 结果、组装宽表，并调用 `step02c` 对 42 个数值去异常值取平均，得到最终 `SFE_ddg`：

```bash
python sfe_fetch.py --config config.json --input prod/sfe_jobs.jsonl --n-frames 21 \
    --wide-output prod/sfe_flex_ddg_wide.csv \
    --run-step02c --sfe-output prod/sfe_scores.csv \
    --step02c-python <装了 numpy/pandas 的 python>
```

> 也可手动单跑聚合脚本：
> ```bash
> python _crpca_download/CRPCA/local_pipeline/step02c_compute_sfe_from_flex_ddg.py \
>   --input prod/sfe_flex_ddg_wide.csv --output prod/sfe_scores.csv
> ```

### 7、汇总得分

用 **`build_pipeline_input.py`** 把 `scores.csv` 与 `sfe_scores.csv` 汇总为序列优化所需的 CSV，并把位置编号修正为 master 序列线性位置：

```bash
python build_pipeline_input.py \
    --scores prod/scores.csv \
    --pdb ab-ag-complex.pdb \
    --antibody-chains M,N \
    --sfe prod/sfe_scores.csv \
    --output prod/single_point_scores.csv
```

CSV 内容说明：

| 列 | 说明 |
|---|---|
| `mutation` | 突变名称，如 `G112E` |
| `chain` | 突变所在链名，须属于 `--antibody-chains`（如 M、N） |
| `location` | 突变所在位置编号（master 序列从 1 开始的线性位置） |
| `original_aa` | WT 残基 |
| `mutant_aa` | 突变残基 |
| `sfe, fep, rosetta_flex, foldx, abnativ` | 五工具原始分数 |

> `--sfe` 可省略（没跑 SFE 时该列留空，`run_pipeline` 会自动跳过该工具）。

## 三、序列优化设计

基于前述单点突变的各个工具评分，进行组合突变枚举与筛选，最终得到优势突变。

```bash
python _crpca_download/CRPCA/local_pipeline/run_pipeline.py \
  --single-point-scores prod/single_point_scores.csv \
  --pdb ab-ag-complex.pdb \
  --antibody-chains M,N \
  --antigen-chains S \
  --output-dir prod/my_run \
  --bo-rounds 3 \
  --num-sequences-per-round 2000 \
  --mei-batch-size 50 \
  --min-mutations 1 \
  --max-mutations 8 \
  --final-quota 20 \
  --seed 0
```

执行参数说明：

| 参数 | 说明 |
|---|---|
| `--single-point-scores` | 单点突变的各工具打分 CSV 文件 |
| `--pdb` | 初始复合物结构，PDB 格式 |
| `--antibody-chains` | 重链,轻链名称，逗号分隔，如 M,N |
| `--antigen-chains` | 抗原链名称 |

带默认值参数：

| 参数 | 默认 | 说明 |
|---|---|---|
| `--bo-rounds` | 3 | 循环枚举突变序列的次数 |
| `--num-sequences-per-round` | 2000 | 每轮生成候选突变数目 |
| `--mei-batch-size` | 50 | 每轮 MEI 保留最佳突变数 |
| `--min-mutations` / `--max-mutations` | 1 / 8 | 组合突变点的范围 |
| `--final-quota` | 20 | 最终推荐突变组合数 |
| `--gp-num-iters` | 500 | GP 训练迭代 |

最终输出：

| 文件 | 内容 |
|---|---|
| `final_recommendations.csv` | 推荐突变组合、mutationHumanReadable、全长 sequence、GP 预测、五工具聚合分 |
| `final_recommendations.fasta` | 同上，FASTA 格式 |
| `pipeline_summary.json` | 运行摘要 |
| `mei_pool_all_rounds.csv` | 所有 BO 轮 MEI 候选合并 |
| `gp_model/` | 训练好的 GP 模型 |

## 附录：运行环境

脚本分两类依赖（建议两个解释器，或一个装齐全部依赖的环境）：

| 用途 | 依赖 | 脚本 |
|---|---|---|
| 提交/回收 WeMol 作业 | `wemol-python-sdk` | prepare_inputs / submit_batch / fetch_batch / sfe_md / sfe_prepare / sfe_fetch |
| 打分聚合 / 序列优化 | `numpy pandas`（step02c）；`biopython==1.79`（build_pipeline_input）；`torch gpytorch scipy scikit-learn`（run_pipeline，CPU 即可） | step02c / build_pipeline_input / run_pipeline |

各打分列对应的 WeMol 模块：

| 打分列 | WeMol 模块 |
|---|---|
| `rosetta_flex` | Flex DDG |
| `foldx` | Mutation Energy of Binding |
| `fep` | Protein FEP |
| `abnativ` | AbnatiV |
| `sfe` | PDB Mutation + Protein MD Simulation v2.1 + MD Trajectory v2 + Flex DDG |
